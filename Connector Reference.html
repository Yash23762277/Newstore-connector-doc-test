Pooling Profile

Field Type Description Default Value Required

Max Active

Number

Controls the maximum number of Mule components that can be borrowed from a session at one time. When set to a negative value, there is no limit to the number of components that may be active simultaneously. If the number of active components exceeds maxActive, the pool is considered exhausted.

Max Idle

Number

Controls the maximum number of Mule components that can remain idle in the pool at any given time. When set to a negative value, there is no limit to the number of idle Mule components.

Max Wait

Number

Specifies the number of milliseconds to wait for a pooled component to become available when the pool is exhausted and the exhaustedAction is set to WHEN_EXHAUSTED_WAIT.

Min Eviction Millis

Number

Determines the minimum amount of time an object can remain idle in the pool before becoming eligible for eviction. If set to a non-positive value, objects will not be evicted from the pool based on idle time alone.

Eviction Check Interval Millis

Number

Specifies the number of milliseconds between runs of the object evictor. If set to a non-positive value, the object evictor will not be executed.

Exhausted Action

Enumeration, one of:

  • WHEN_EXHAUSTED_GROW

  • WHEN_EXHAUSTED_WAIT

  • WHEN_EXHAUSTED_FAIL

Specifies the behavior of the Mule component pool when it becomes exhausted. The possible values are: "WHEN_EXHAUSTED_FAIL": Throws a NoSuchElementException. "WHEN_EXHAUSTED_WAIT": Blocks by invoking Object.wait(long) until a new or idle object is available. If maxWait is positive, it blocks for at most that many milliseconds; otherwise, it blocks indefinitely if maxWait is negative. "WHEN_EXHAUSTED_GROW": Creates a new Mule instance and returns it, essentially ignoring maxActive.

Initialisation Policy

Enumeration, one of:

  • INITIALISE_NONE

  • INITIALISE_ONE

  • INITIALISE_ALL

Determines how components in a pool should be initialized. The options are: - INITIALISE_NONE: Does not load any components into the pool on startup. - INITIALISE_ONE: Loads one initial component into the pool on startup. - INITIALISE_ALL: Loads all components into the pool on startup.

Disabled

Boolean

Whether pooling should be disabled

Repeatable In Memory Stream

Field Type Description Default Value Required

Initial Buffer Size

Number

This attribute specifies the initial amount of memory allocated to consume the stream and enable random access. If the stream exceeds this buffer size, it can be expanded based on the `bufferSizeIncrement` attribute, with a maximum limit set by `maxInMemorySize`.

Buffer Size Increment

Number

This attribute determines by how much the buffer size will be expanded if it exceeds its initial size. Setting a value of zero or lower means that the buffer should not expand, and a STREAM_MAXIMUM_SIZE_EXCEEDED error will be raised when the buffer reaches its full capacity.

Max Buffer Size

Number

This attribute defines the maximum amount of memory that can be utilized. If the memory usage exceeds this limit, a STREAM_MAXIMUM_SIZE_EXCEEDED error will be raised. Setting a value equal to or lower than zero implies there is no memory limit.

Buffer Unit

Enumeration, one of:

  • BYTE

  • KB

  • MB

  • GB

The unit in which all these attributes are expressed
